Record Property

Record Property is a plugin for CSS, which gets a record name from a text field or context menu of another CSS plugin and displays all possible informations, that can be gathered for that record.

Basic logic

  • connect to RDB (Oracle)
    • if works
      • get record type from DAL (or from record name, if first one fails)
      • get data from RDB, fill first and second column
      • get data from DAL, fill third column
      • get data from RMI (IOC), fill fourth column (note this one is usually empty), fill fourth column
    • if doesn't work
      • get record type from DAL (or from record name, if first one fails)
      • get field names from RDB for that type, fill first column, second stays empty
      • get data from DAL, fill third column
      • get data from RMI (IOC), fill fourth column (note this one is usually empty), fill fourth column
RMI(IOC) is accessed through SaveButton plugin.

If record name is not valid?

Plugin deletes last part of name, if it has dot and less than 6 characters (for example: ".abcd).

If record is not found at all, plugin will just print empty table.

If no data?

If data is not available or accessible for a single cell, it displays a gray-yellow background and "N/A" message.

If data for cell, that was gotten, is empty, it does not display "N/A" message, but just empty string.

Tips and tricks

To find a field name quickly, just type it and it will automatically go to a row with that field name.

Column names can be changed easily in a file "messages.properties".
