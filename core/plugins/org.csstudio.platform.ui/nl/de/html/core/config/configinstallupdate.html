Option

Description

Default

Maximale Anzahl von 'History'-Konfigurationen

Maximale Anzahl von Konfigurationen, die man in der Konfigurations-History behalten möchte. Sie werden aufgehoben, damit man zu einer vorigen Konfiguration zurückkehren kann.

100

Checken digitaler Signaturen von Download-Archiven

Diese Option kann digitaler Signaturen von Download-Archiven checken.

On

Automatisch ausgewählte 'mirrors'

Wenn diese Variable auf 0 gesetzt ist, wird man bei jedem Update aufgefordert, einen Server auszuwählen, von welchem die Updates dann heruntergeladen werden.

Off

Gültige Updates

Angenommen, dass man die Form 'major.minor.service' benutzt, kann man den zu verwendenden Update-Level auswählen aus:
equivalent
Nur Service Updates werden angezeigt.
compatible
Service updates und minor updates werden angezeigt.

equivalent

Update-Verfahren

Der Update-Verfahrens-URL, der die Umleitung von Update-Seiten innerhalb einer Organisation steuert.

No policy
