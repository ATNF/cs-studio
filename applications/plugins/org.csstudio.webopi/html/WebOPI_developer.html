WebOPI Developer Manual

Required Plugins

WebOPI is developed on RAP, so you must have RAP installed in your Eclipse. You can simple download Eclipse for RCP and RAP developers from http://www.eclipse.org/downloads/. WebOPI also based on the RAP GEF which is an incubator project under RAP, so you need to get org.eclipse.draw2d, org.eclipse.draw2d.rap and org.eclipse.gef from RAP CVS: http://www.eclipse.org/rap/source/.

Launch WebOPI from Eclipse

Open plugin.xml in org.csstudio.webopi plugin and click Launch a RAP application. Since there is no $CATALINA_HOME is set, the preferences will not be loaded.

Build WAR file

Open webopi.warproduct and click on Eclipse WAR product export wizard to export the WAR file. To allow multiple sessions in a same browser, you need to disable cookies. Copy the folder META-INF under html foder to the same directory of webopi.war. Then execute the command jar uf webopi.war META-INF/ to add this folder to webopi.war. Now, you can deploy the WAR file on a tomcat server. See WebOPI user manual.

Debug URL

To show debug information, you can append rap?startup=debug to webopi/. For example: http://your_site.com/webopi/rap?startup=debug. If you want to show more lines, you can append &lines=number. For example, http://your_site.com/webopi/rap?startup=debug&lines=500.
