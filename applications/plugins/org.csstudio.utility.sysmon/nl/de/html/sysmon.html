CSS-System-Monitor

Ein einfacher System-Monitor zum Überwachen der Speicherbenutzung.

Die angezeigten "max", "total" und "free" Speicherwerte werden von der Java- Runtime-API übernommen. Ihre Bedeutung ist nicht unmittelbar einleuchtend:

  • "Max" Memory:
    Der maximale Betrag an Speicher, den die Java Virtual Machine (JVM) je zu benutzen versuchen wird. Dies ist eine Obergrenze, die sicherstellt, dass JVM nicht den gesamten Speicher belegt.
  • "Total":
    Der Betrag an Speicher, den die JVM aktuell für "objects in-use" in Benutzung hat und für neue Objekte bereits reserviert hat. Viele Leute nennen diesen Speicher "Available" Memory.
  • "Free":
    Speicher, den die JVM als "frei" betrachtet.
Wenn man einfach "available" and "used" Memory betrachtet, könnte man annehmen, dass "max - total + free" gleich dem verfügbaren Speicher ist, aber das Verhalten der JVM Garbage-Collection ist sehr unvorhersagbar: Manchmal wird Speicher festgehalten, der eigentlich schon frei sein sollte und manchmal wird er sofort freigegeben.

Manuelle Garbage-Collection

Die Betätigung des "GC"-Knopfs triggert eine Garbage Collection. Das kann für Tests sinnvoll sein, aber im Allgemeinen ist es am Besten, die Garbage-Collection der JVM zu überlassen; d.h., es ist nicht nötig, dass Benutzer periodisch die Garbage-Collection triggern.

Setup

Alle Einstellungen können über die System-Monitor-Preference-Page verändert werden.
